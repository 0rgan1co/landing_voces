Gaps detectados

  • Cookie banner no persiste la decision (sin localStorage)
  • Selector de idioma no funciona
  • Link "Precios" no lleva a ninguna seccion
  • No hay boton "Entrar" en navbar para usuarios existentes
  • No hay Google Analytics integrado (el cookie banner lo menciona pero no lo carga)
  • Demo section tiene placeholders (screenshots vacios)
  • No hay favicon, OG tags ni structured data
  • No hay autenticacion ni acceso a dashboard
  • CTAs de "Agendar Demo" no conectan a Calendly/Cal.com
  • WhatsApp CTA no tiene link real

A. Autenticacion y Acceso al Dashboard

IDPatternRequirement
EARS-001UbiquitousThe landing page shall display a "Entrar" button in the navbar, positioned before the CTA "Agendar Demo".
EARS-002Event-DrivenWhen the user clicks "Entrar", the system shall display a sign-in modal with a "Continuar con Google" button as the only authentication method.
EARS-003Event-DrivenWhen the user clicks "Continuar con Google", the system shall initiate the Google OAuth 2.0 authorization flow using the Authorization Code grant type with PKCE.
EARS-004Event-DrivenWhen Google returns a valid authorization code, the system shall exchange it for access and refresh tokens via the backend token endpoint.
EARS-005Event-DrivenWhen the token exchange succeeds, the system shall store the session token in an HttpOnly secure cookie and redirect the user to /dashboard.
EARS-006UnwantedIf the Google OAuth flow fails or the user cancels, then the system shall display the message "No se pudo iniciar sesion. Intenta de nuevo." and return the user to the landing page.
EARS-007UnwantedIf the user's email domain is not in the authorized organizations list, then the system shall display "Tu organizacion no tiene acceso. Contacta a tu administrador." and deny access to the dashboard.
EARS-008State-DrivenWhile the user has an active session, the navbar shall replace the "Entrar" button with the user's avatar and name, and link to /dashboard.
EARS-009Event-DrivenWhen the user clicks their avatar in the navbar, the system shall display a dropdown with "Ir al Dashboard" and "Cerrar sesion" options.
EARS-010Event-DrivenWhen the user clicks "Cerrar sesion", the system shall revoke the Google token, delete the session cookie, and redirect to the landing page.
EARS-011State-DrivenWhile the user's session token has expired, the system shall attempt a silent token refresh using the stored refresh token before prompting re-authentication.
EARS-012UnwantedIf the silent token refresh fails, then the system shall clear the session and redirect the user to the landing page with the sign-in modal open.

B. Cookie Banner y Consentimiento

IDPatternRequirement
EARS-013Event-DrivenWhen the user clicks "Aceptar" on the cookie banner, the system shall store the consent preference in localStorage with key voces_cookie_consent and value accepted.
EARS-014Event-DrivenWhen the user clicks "Rechazar" on the cookie banner, the system shall store the consent preference in localStorage with key voces_cookie_consent and value rejected.
EARS-015ComplexWhile voces_cookie_consent exists in localStorage, when the page loads, the system shall hide the cookie banner.
EARS-016ComplexWhile voces_cookie_consent equals accepted, when the page loads, the system shall load the Google Analytics gtag.js script with the configured measurement ID.
EARS-017ComplexWhile voces_cookie_consent equals rejected, when the page loads, the system shall not load any tracking scripts.

C. Internacionalizacion (i18n)

IDPatternRequirement
EARS-018Event-DrivenWhen the user selects a language from the language selector, the system shall reload the page content in the selected language (ES, EN, or PT).
EARS-019Event-DrivenWhen the user selects a language, the system shall store the preference in localStorage with key voces_lang.
EARS-020ComplexWhile voces_lang exists in localStorage, when the page loads, the system shall render content in the stored language.
EARS-021UbiquitousThe system shall set the <html lang> attribute to match the active language.

D. Navegacion y CTAs Funcionales

IDPatternRequirement
EARS-022Event-DrivenWhen the user clicks "Agendar Demo" (any instance), the system shall open a scheduling widget (Calendly or Cal.com) in a modal overlay.
EARS-023Event-DrivenWhen the user clicks "Escribinos por WhatsApp", the system shall open https://wa.me/{numero} with a pre-filled message in a new tab.
EARS-024Event-DrivenWhen the user clicks the "Precios" nav link, the system shall scroll to the pricing section of the page.
EARS-025UbiquitousThe landing page shall include a pricing section (#precios) with plan comparison between the "Agendar Demo" and "Entrar" CTAs.

E. Demo / Product Tour

IDPatternRequirement
EARS-026Event-DrivenWhen the user clicks a demo link ("Entrevistas con IA", "Extraccion de Narrativas", or "Mapa Semantico"), the system shall display the corresponding screenshot in the main gallery area.
EARS-027Event-DrivenWhen the user clicks a thumbnail in the demo gallery, the system shall replace the main screenshot with the selected thumbnail's full-size image.
EARS-028UbiquitousThe demo section shall display real product screenshots, not placeholder elements.

F. SEO, Performance y Metadata

IDPatternRequirement
EARS-029UbiquitousThe landing page shall include Open Graph meta tags (og:title, og:description, og:image, og:url) for link previews on social platforms.
EARS-030UbiquitousThe landing page shall include a favicon in SVG and PNG formats.
EARS-031UbiquitousThe landing page shall include structured data (JSON-LD) with Organization and SoftwareApplication schema types.
EARS-032UbiquitousThe landing page shall load the Inter font from Google Fonts or serve it as a self-hosted WOFF2 file.
EARS-033UbiquitousThe landing page shall achieve a Lighthouse Performance score of 90 or higher on mobile.

G. Seguridad

IDPatternRequirement
EARS-034UbiquitousThe system shall set the following HTTP headers on all responses: X-Content-Type-Options: nosniff, X-Frame-Options: DENY, Referrer-Policy: strict-origin-when-cross-origin.
EARS-035UbiquitousThe Google OAuth client ID shall be stored as an environment variable, not hardcoded in client-side code.
EARS-036UbiquitousThe system shall validate the Google ID token's iss, aud, and exp claims on the backend before creating a session.

  5. Bloque E — Demo tour (requiere screenshots reales)
  6. Bloque C — i18n (expansion regional)
