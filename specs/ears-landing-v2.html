A. Autenticacion y Acceso al Dashboard

CumpleIDPatternRequirementEstado
EARS-001UbiquitousThe landing page shall display a "Entrar" button in the navbar, positioned before the CTA "Agendar Demo".
EARS-002Event-DrivenWhen the user clicks "Entrar", the system shall display a sign-in modal with a "Continuar con Google" button as the only authentication method.
EARS-003Event-DrivenWhen the user clicks "Continuar con Google", the system shall initiate the Google OAuth 2.0 authorization flow using the Authorization Code grant type with PKCE.
EARS-004Event-DrivenWhen Google returns a valid authorization code, the system shall exchange it for access and refresh tokens via the backend token endpoint.
EARS-005Event-DrivenWhen the token exchange succeeds, the system shall store the session token in an HttpOnly secure cookie and redirect the user to /dashboard.
EARS-006UnwantedIf the Google OAuth flow fails or the user cancels, then the system shall display the message "No se pudo iniciar sesion. Intenta de nuevo." and return the user to the landing page.
EARS-007UnwantedIf the user's email domain is not in the authorized organizations list, then the system shall display "Tu organizacion no tiene acceso. Contacta a tu administrador." and deny access to the dashboard.
EARS-008State-DrivenWhile the user has an active session, the navbar shall replace the "Entrar" button with the user's avatar and name, and link to /dashboard.
EARS-009Event-DrivenWhen the user clicks their avatar in the navbar, the system shall display a dropdown with "Ir al Dashboard" and "Cerrar sesion" options.
EARS-010Event-DrivenWhen the user clicks "Cerrar sesion", the system shall revoke the Google token, delete the session cookie, and redirect to the landing page.
EARS-011State-DrivenWhile the user's session token has expired, the system shall attempt a silent token refresh using the stored refresh token before prompting re-authentication.
EARS-012UnwantedIf the silent token refresh fails, then the system shall clear the session and redirect the user to the landing page with the sign-in modal open.

CumpleIDPatternRequirementEstado

EARS-085State-DrivenMientras una narrativa posea una probabilidad de pertenencia inferior al threshold definido, el sistema debera clasificarla como "atipica" y visualizarla en color gris.
